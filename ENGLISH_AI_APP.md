# English AI Learning App

هذا الفرع مخصص لتطبيق تعلم اللغة الإنجليزية بالذكاء الاصطناعي، وهو منفصل عن فرع `main` حتى لا يتأثر مشروع غرفة الطوارئ.

## المراحل المنجزة

### المرحلة الأولى

- التسجيل وتسجيل الدخول عبر Supabase Auth.
- إعداد ملف المتعلم والهدف واللهجة والاهتمامات والوقت اليومي.
- NestJS API مرتبط بـ PostgreSQL/Supabase.

### المرحلة الثانية

- اختبار تحديد مستوى تكيفي CEFR من A1 إلى C2.
- تقييم مستقل للقواعد والمفردات والقراءة والاستماع.
- تخزين كل جلسة وإجابة ونتيجة داخل قاعدة البيانات.

### المرحلة الثالثة

- صفحة رئيسية ذكية ومستوى مستقل لكل مهارة.
- هدف يومي وXP وسلسلة تعلم ومراجعات مستحقة.
- مسار أسبوعي شخصي من 7 إلى 14 نشاطاً.
- خوارزمية تراعي نقاط الضعف والهدف والاهتمامات.

### المرحلة الرابعة

- محرك دروس تفاعلي قابل للحفظ والاستئناف.
- تسعة أنواع من الخطوات: شرح، مفردات، قراءة، استماع، اختيار متعدد، صح/خطأ، ملء فراغ، كتابة موجهة، وخلاصة.
- نص صوتي متزامن على مستوى المقاطع.
- تصحيح الإجابات على الخادم دون إرسال مفاتيح الإجابات إلى Flutter.
- حفظ كل محاولة وزمنها ودرجتها في PostgreSQL.
- نسخة ثابتة من محتوى الدرس لكل جلسة لمنع تغيره أثناء التعلم.
- إكمال النشاط وإضافة XP وتحديث السلسلة وفتح النشاط التالي تلقائياً.
- دروس أولية فعلية لجميع مستويات A1–C2 والمهارات الست.
- Supabase migration رقم `202606260004_interactive_lessons.sql`.

## التحقق

- NestJS build: ناجح.
- اختبارات الخادم: 12 من 12 ناجحة.
- Production dependency audit: لا توجد ثغرات مسجلة.
- فحص بنية واستيرادات ملفات Dart: ناجح.
- فحص سلامة الحزم المضغوطة: ناجح.

تعذر تشغيل `flutter analyze` و`flutter test` داخل بيئة الإنشاء لعدم توفر Flutter SDK. تمت إضافة اختبار نماذج Flutter إلى المشروع ليعمل عبر CI أو جهاز التطوير.

## ربط Supabase عبر GitHub Actions

أضف الأسرار التالية من إعدادات المستودع:

- `SUPABASE_ACCESS_TOKEN`
- `SUPABASE_PROJECT_REF`
- `SUPABASE_DB_PASSWORD`

لا ترفع كلمة مرور قاعدة البيانات أو Service Role Key أو أي مفتاح سري داخل الملفات.

النسخة الكاملة للمرحلة الرابعة محفوظة في الحزمة المرفقة داخل محادثة المشروع. ويظل إنشاء مستودع مستقل باسم `english-ai-learning-app` أفضل خيار عند بدء النشر الفعلي.
